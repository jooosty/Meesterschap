De opdracht

Het web is voor iedereen. Dat betekent dat je website moet werken op oude browsers, nieuwe browsers, telefoons, game consoles, met screenreaders, met een muis, met alleen een toetsenbord en alles daartussenin.

De eindopdracht: zet een officiële belasting-PDF om naar een interactief webformulier. Aangifte erfbelasting een van de meest complexe formulieren die burgers moeten invullen. Uitgangspunt: HTML first, dan CSS, dan JavaScript.

Wat ik heb gebouwd

Een volledig meerstappe formulier voor de aangifte erfbelasting 2025, gebaseerd op het officiële Belastingdienst-formulier. Met real-time validatie, voortgangsbalk, print-styling en donkere modus. De stijl is gebaseerd op het officiële NS design system.

Code highlight: elfproef BSN-validatie

// Elfproef-validatie voor BSN (weging 9-8-7-6-5-4-3-2-1)
const elfValidBsn = (bsn) => {
  if (!/^\d{8,9}$/.test(bsn)) return false;
  const digits = bsn.padStart(9, "0").split("").map(Number);
  const sum = digits.reduce((acc, digit, index) => {
    const weight = index === 8 ? -1 : 9 - index;
    return acc + digit * weight;
  }, 0);
  return sum % 11 === 0;
};

Code highlight: inline validatie met aria-live

// Feedback span zoeken of aanmaken naast het veld
const getFeedback = (el) => {
  let next = el.nextElementSibling;
  while (next) {
    if (next.classList.contains("inline-feedback")) return next;
    next = next.nextElementSibling;
  }
  const feedback = document.createElement("span");
  feedback.className = "inline-feedback";
  feedback.setAttribute("aria-live", "polite");
  el.insertAdjacentElement("afterend", feedback);
  return feedback;
};

Code highlight: localStorage concept-opslag

const saveFormDraft = () => {
  const draft = {};
  formElement.querySelectorAll("input, select, textarea").forEach((field) => {
    if (!field.name || field.type === "file") return;
    if (field.type === "radio") {
      if (field.checked) draft[field.name] = field.value;
      return;
    }
    draft[field.name] = field.value;
  });
  localStorage.setItem("erfbelasting-form-draft", JSON.stringify(draft));
};

Nieuwe inzichten

01

Formulieren zijn complexer dan je denkt

Als je een écht formulier bouwt met conditionele vragen, validatie op meerdere niveaus en accessibility merk je hoe veel er mis kan gaan.

02

HTML biedt meer dan je gebruikt

fieldset, legend, datalist, dialog, autocomplete, inputmode native features die UX verbeteren zonder JavaScript.

03

Validatie is een UX-beslissing

Wanneer toon je een foutmelding? Direct bij typen, bij blur, bij submit? Elke keuze heeft gevolgen voor hoe frustrerend of behulpzaam het formulier aanvoelt.

04

localStorage als no-cost feature

Automatisch concept opslaan kost bijna geen code maar levert enorm veel UX op. Niemand wil een lang formulier opnieuw beginnen.

05

Een design system gebruiken leert je patronen

Het NS design system omzetten naar CSS tokens leerde me hoe je afstand en kleur systematisch organiseert.

06

SVG voor custom UI elementen

Voor de verbindingslijnen in de voortgangsbalk had ik inline SVG nodig nieuw voor mij en precies genoeg voor dit doel.

Reflectie

Browser Technologies was het vak waarbij ik het meest verrast werd door mezelf. Aan het begin dacht ik formulieren al te begrijpen. Na zes weken bouwen aan één complex formulier besef ik hoe naïef dat was.

De elfproef was een moment waarop techniek en werkelijkheid samenkwamen. Een algoritme van papier naar werkende code brengen en het dan ook écht zien werken met testgetallen van de Belastingdienst was bevredigend op een manier die ik niet had verwacht.

De 10 is mooi. Maar de feedback dat ik nu écht begrijp hoe complex formulieren zijn voelt als de eigenlijke beloning.

Learning goals

  • Learn how to effectively use CSS: Bij browser-tech viel het belang van toegankelijke focus-styling op. Ik ga consistente focus- en form-styles als basis opnemen in mijn componenten. Dit verbetert navigatie voor toetsenbordgebruikers.
  • Learn how to animate using CSS: Transities voor validatie en input-feedback werkten goed en maakten interacties duidelijk. Ik ontwerp minimalistische feedback-animaties en respecteer prefers-reduced-motion. Dat houdt interfaces snel en toegankelijk.
  • Be more creative in designing: Heldere composities bleken gebruikers te helpen bij het scannen van content. Ik experimenteer met micro-interacties die de bruikbaarheid verhogen. Kleine bewegingen ondersteunen context zonder te storen.
