Sprint
Zero

Een persoonlijke website bouwen die je informatie verbergt achter games. Speel Tetris, Minesweeper of 2048 om te ontgrendelen wie ik ben.

Deadline

12 FEB

Duur

8 dagen

Uren

~32u

Beoordeling

VOLDAAN

De opdracht

Maak een persoonlijke website die zichzelf onthult naarmate bezoekers punten verdienen. Informatie over mij naam, leeftijd, locatie, hobbies, doelen is vergrendeld achter een gamescore. Hoe meer je speelt, hoe meer je ontgrendelt.

Wat ik heb gebouwd

In twee weken en ~32 uur groeide de site van een simpele Tetris-clone naar een volledige mini-arcade met drie games, easter eggs, een leaderboard en een mobiele versie.

Tetris

Volledig werkend, met difficulty slider, speedup toggle, piece saving, afbeeldingen als blokken en een mobiele besturing.

Minesweeper

Klassiek Minesweeper met instelbaar grid-formaat en mijnendichtheid. Punten op basis van succesvol vrijgelegde vakjes.

2048

Schuivende tegel-puzzel met animatie-feedback. Score telt mee voor het totaal en activeert unlock drempelwaarden.

Leaderboard

Scores worden opgeslagen in localStorage. Top 10 per game, te openen via knop of door lb te typen.

Dark / light mode

Keuze wordt opgeslagen en respecteert ook de systeeminstelling als de gebruiker nog geen keuze heeft gemaakt.

Easter eggs

Een auto rijdt langs het scherm, er zijn keyboard-sequenties verborgen voor barrel rolls, meteorenregen en meer.

Code highlight: gedeelde score

Het lastigste was drie losse games één gecombineerde unlock-score laten bijhouden. Ik heb een centrale sharedScoreState gemaakt in main.js die elk spel aanroept via een globale functie: 

// main.js centrale score state voor 3 games
const sharedScoreState = {
  tetris: 0, minesweeper: 0, '2048': 0, total: 0
};

window.setSharedScore = function(value, source) {
  if (!source || !(source in sharedScoreState)) return;
  sharedScoreState[source] = normalizeSharedScore(value);
  updateSharedTotal(); // herberekent + triggert unlock
};

Elk spel roept window.setSharedScore(score, 'tetris') aan als zijn score verandert. De totaalteller update live en checkt welke about-me-secties ontgrendeld moeten worden.

Code highlight: easter eggs via keysequenties

De easter eggs werken door keysequenties te tracken. Elke reeks heeft een eigen index die reset na 1,2 seconden inactiviteit. Typ roll voor een barrel roll, of de Konami code voor een meteorenregen: 

// effects.js barrel roll via "roll" typen
const rollSequence = ['r', 'o', 'l', 'l'];
let rollIndex = 0;

function handleRollSequence(key) {
  if (key === rollSequence[rollIndex]) {
    rollIndex += 1;
    if (rollIndex === rollSequence.length) {
      triggerBarrelRoll(); // 🌀 roteer de hele pagina
      rollIndex = 0;
    }
  } else {
    rollIndex = key === rollSequence[0] ? 1 : 0;
  }
}

Alle verborgen commando's:

roll → barrel roll scan → scanline sweep lb → leaderboard picture → foto galerij test → smoke tests Konami → meteorenregen Z vasthouden → slow motion

Nieuwe inzichten

01

CSS variabelen voor animaties

CSS custom properties zijn krachtig buiten kleurgebruik. Via setProperty in JS kun je animaties sturen zonder classes te wisselen ik gebruikte dit voor de 2048 slide-richting.

02

Responsive zonder framework

Met CSS Grid, clamp() voor fluid type en een handvol media queries is een mobiele versie prima te bouwen ook voor interactieve games met touch controls.

03

State management in vanilla JS

Drie games, één totaalscore. In plaats van een framework heb ik een globaal pub/sub patroon gemaakt via window.setSharedScore. Simpel, makkelijk uitbreidbaar en precies genoeg. Dit is wat ik wil blijven verdiepen.

04

Leesbaarheid van CSS

Na de code review begreep ik hoe whitespace en naamgeving in CSS de leesbaarheid enorm verbeteren. Code schrijf je ook voor andere mensen, niet alleen de browser.

Reflectie

Sprint 0 was meteen een diepe duik in zelf bouwen. Drie games tegelijk schrijven die ook op mobiel werken en een gecombineerde score bijhouden dat was meer dan verwacht.

Het meest geleerd van het organiseren van een project terwijl het groeit. Alles begon in één bestand. Zodra dat te onoverzichtelijk werd, heb ik gesplitst per verantwoordelijkheid. Dat maakte debuggen en uitbreiden aanzienlijk makkelijker.

De feedback dat leerdoel 3 het meest interessant was, bevestigt iets wat ik zelf ook voelde: state management zonder framework is het soort low-level begrip dat ik wil verdiepen. Wanneer heb je echt een library nodig, en wanneer kom je met vanilla JS verder dan je denkt? Dat wordt een terugkerende vraag in de rest van de minor.

Learning goals

  • Learn how to effectively use CSS: Tijdens Sprint-0 heb ik layout- en responsiviteitskeuzes getest voor kleine interfaces. Ik ga breakpoints en component-gedrag systematiseren zodat toekomstige iteraties sneller zijn. Dit draagt bij aan consistente weergave over devices.
  • Learn how to animate using CSS: Animaties gaven waardevolle state-feedback bij buttons en toasts. Ik houd animaties voortaan subtiel en voeg checks voor prefers-reduced-motion toe. Zo blijft de ervaring toegankelijk en voorspelbaar.
  • Be more creative in designing: De prototype-stadia nodigen uit tot speelse micro-interacties. Daarom plan ik korte UX-experimenten per feature. Resultaten worden meegenomen naar de volgende iteratie om bruikbaarheid te verbeteren.
