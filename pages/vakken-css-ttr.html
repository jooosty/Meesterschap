Sector 05a CSS to the Rescue

CSS to the
Rescue

Een volledig functioneel kerncentrale bedieningspaneel gebouwd met bijna uitsluitend CSS. Geen JavaScript voor interactie, geen ID- of class-selectors voor styling alleen de kracht van CSS.

9,5 / 10

Vak

CSS TTR

Concept

KRAFTWERK

Uren

~24u

Score

9,5 / 10

De opdracht

Maak een interactief component met bijna uitsluitend CSS. De twee kernregels: geen IDs of classes als styling-hooks (alleen voor :target en <label for>), en geen JavaScript voor logica alleen een minimale range-to-custom-property bridge was toegestaan.

Het doel: de grenzen van CSS opzoeken. Alles wat je normaal met JavaScript zou doen, moet opgelost worden met selectors, pseudo-classes en de cascade.

Het concept: KRAFTWERK

Ik koos voor een fictief Duits kerncentrale bedieningspaneel, geïnspireerd op fotografieën van verlaten Oost-Europese controlekamers. De naam KRAFTWERK Duits voor "energiecentrale" paste perfect bij de retro-industriële esthetiek.

Het paneel bevat vier analoge meters, zes LED-indicatoren, drie toggleschakelaars, vier draaiknoppen, START/STOP besturing en een retro scroll-drum MW display. Alle interacties zijn puur CSS ze zijn onderling verbonden: schakel je een pomp uit terwijl de centrale draait, dan flash het schakelpaneel oranje.

Componenten

Analoge meters × 4

Frequency, Pressure, Temperature, Power. Naald rotatie aangestuurd via CSS custom property van de knob-range input. Werkt alleen als de centrale actief is.

Toggle switches × 3

Pump A, Pump B, Turbine. Pure input[type=checkbox] met gestylede label. Schakelen ze uit tijdens het draaien, dan activeert een oranje waarschuwingsflash.

Draaiknoppen × 4

input[type=range] gestyled als roterende knop. JS schrijft alleen de knob-waarde naar een CSS custom property. Rotatie, kleur en alle feedback: puur CSS.

LED indicatoren × 6

Reageren op de toggle switches via de ~ sibling combinator en :not(:checked).

START / STOP

Anchor links die :target activeren op verborgen <div> elementen. De hele interactieve staat van het paneel hangt hier aan.

MW Scroll display

Retro "drum"-display gebouwd met 21 <span>-elementen en een scroll-animatie. Flikkert rood bij een meltdown.

EMERGENCY meltdown

Alle vier knoppen op maximum + centrale actief = EMERGENCY overlay. Volledig via container style queries. Het scherm schudt.

Easter eggs × 5

Inspectie-stempel, ingenieursgravure, koffievlek, versleten pomp-label, en een plantnaam die verschijnt bij 2 seconden hoveren op de titel.

Code highlight: container style queries voor meltdown

Als alle vier knoppen op 145 staan én de centrale draait, treedt er een noodtoestand op. Dit vereist het checken van vier CSS custom property waarden tegelijk normaal JavaScript werk. De oplossing: geneste container style queries: 

/* state.css meltdown triggert alleen als alle 4 knoppen op 145 staan */
@container style(--knob-power: 145) {
  @container style(--knob-freq: 145) {
    @container style(--knob-pressure: 145) {
      @container style(--knob-temp: 145) {
        #running:target ~ main > p:last-of-type {
          opacity: 1;
          animation: emergency-pulse 0.8s ease-in-out infinite;
        }
        #running:target ~ main {
          animation:
            meltdown-shake 0.15s ease-in-out infinite,
            overload-flash  0.6s ease-in-out infinite;
        }
      }
    }
  }
}

Code highlight: letter-corrupt animatie

De letters van KRAFTWERK flikkeren onregelmatig en wisselen kort van glyph alsof er een elektrische storing is. Elke letter heeft zijn eigen timing via per-letter CSS variabelen: 

/* K */ header > h1 > span:first-child > span:nth-child(1) {
  --glyph:            'К';
  --corrupt-duration: 11.3s;
  --corrupt-delay:    1.2s;
  --flicker-duration: 2.8s;
}
header > h1 > span > span {
  animation:
    letter-stamp-in  0.4s cubic-bezier(0.22, 1, 0.36, 1) var(--stamp-delay) forwards,
    letter-flicker   var(--flicker-duration) var(--flicker-delay) ease-in-out infinite,
    letter-hide      var(--corrupt-duration) var(--corrupt-delay) steps(1, end) infinite;
}

Code highlight: knob → meter binding

De enige JavaScript schrijft de waarde van input[type=range] naar een CSS custom property. CSS pakt die op en berekent de rotatie van de naald: 

/* Het enige JS range → custom property bridge */
const updateRange = range => {
  document.documentElement.style.setProperty('--' + range.name, range.value);
};

/* CSS pakt de waarde op */
#running:target ~ main > section:first-of-type
  > figure:nth-of-type(1) > div > span:nth-child(3) {
  transform: translateX(-50%) rotate(calc(var(--knob-freq) * 0.48deg));
}

Experimenten die niet werkten

× Geluidseffecten

CSS heeft geen audio capabilities kliksgeluiden bij schakelaars zijn fundamenteel onmogelijk zonder JS.

× CSS-only timer

Een lopende timer integreren met de :target state bleek niet correct werkbaar.

× Willekeurige startwaarden

CSS heeft geen random-functie die beginwaarden van range inputs kan instellen.

Nieuwe inzichten

01

CSS kan meer interactiviteit dan je denkt

Draaiknoppen, conditional waarschuwingen, een meltdown zonder logica-JS. Denk in state via inputs en pseudo-classes, niet in event listeners.

02

Container style queries = CSS conditional logic

Custom property waarden lezen via @container style() maakt CSS conditioneel op basis van data niet alleen afmetingen.

03

Selector specificity als architectuurtool

Zonder classes navigeer je bewust met ~, :nth-of-type() en :not(). De HTML-structuur is je ontwerpdocument.

04

@layer voor cascade-controle

CSS in lagen organiseren stuurt de cascade expliciet zonder te vertrouwen op specificity of importvolgorde.

Reflectie

CSS to the Rescue was het vak waarbij ik het meest verrast werd door CSS zelf. Een volledig interactief bedieningspaneel waarbij knopwaarden meters aansturen, switches LEDs doden en vier maxen een noodalarm triggert puur via de cascade had ik niet voor mogelijk gehouden.

Het concept hielp enorm. Door een duidelijk thema te kiezen kreeg elk CSS-probleem een narratieve context. De koffievlek is niet zomaar een cirkel; de worn-off Pump B label is iets wat je in een echte verlaten controlekamer zou vinden. Elke technische beslissing werd ook een ontwerpbeslissing.

De 9,5 voelt verdiend maar ook als aanmoediging. Eerder structureren scheelt tijd de file-restructuring kostte vier uur die ik vroeger had kunnen besparen.

Learning goals

  • Learn how to effectively use CSS: Ik merkte dat het structureren van CSS (component-classes, logische properties, container-queries) mijn workflow versnelt. Daarom ga ik deze patronen consequent toepassen in volgende projecten. Dit voorkomt veel herwerk en maakt onderhoud eenvoudiger.
  • Learn how to animate using CSS: Voor kleine UI-reveal-animaties gebruikte ik keyframes en transities. Voortaan check ik altijd prefers-reduced-motion en bied ik een statische fallback. Zo blijft de ervaring voor iedereen betrouwbaar.
  • Be more creative in designing: Door te spelen met compositie en kleur ontdekte ik interessante hiërarchie-opties. Ik zal deze experimenten vastleggen als kleine component-varianten. Die varianten kunnen later makkelijk hergebruikt worden in andere projecten.
