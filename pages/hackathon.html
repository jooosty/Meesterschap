De opdracht

In één week bouwen we als team een prototype voor een echte opdrachtgever: SRON — het Netherlands Institute for Space Research. De challenge: maak een website voor NEBULA Xplorer, een fictieve maar realistische satellietmissie om zwarte gaten te bestuderen met een X-ray telescoop.

Eisen: goede UX én over the top. Spectaculaire styling én over the top. Responsive, toegankelijk, elegante code, oog voor detail — en paaseieren verplicht. Geen frameworks, native web technologie first.

Het concept: NEBULA Xplorer

NEBULA staat voor Netherlands Educational Satellite for Exploration of Binary-Linked Astrophysics – X-ray Observer. ~400 studenten van 14 Nederlandse onderwijsinstellingen helpen SRON om deze satellietmissie van begin tot eind te bouwen.

Onze site vertelt dit verhaal: wetenschappelijke missie, instrumenten van de satelliet, crew-pagina, stageplaatsen en contactformulier — allemaal in een donkere, immersieve ruimte-esthetiek met oranje accenten.

Cursor mode systeem

  • Vijf cursor-modi via cijfertoetsen 1–5: Astronaut, Comet, Black Hole, Laser, Supernova
  • Elke modus heeft een eigen SVG cursor én canvas-gebaseerd particle systeem
  • Astronaut: volgt de muis met vertraagd "drijvend in de ruimte" gevoel
  • Comet: laat een stofspoor achter in oranje en cyaan
  • Black Hole: particles worden van buiten naar binnen gezogen
  • Laser: trekt een gloeiende cyane lichtstreep achter de muis
  • Supernova: klik om 50 particles te exploderen in alle richtingen
  • Automatisch uitgeschakeld op touchscreens via matchMedia('(pointer: coarse)')
  • Toast-notificatie bij elke moduswissel

Easter egg: black hole klikken

  • Klik 10× op het zwarte gat op de homepage → animated accretie-schijf verschijnt als canvas overlay
  • Klik nog 10× (totaal 20) → portret van docent Sanne verschijnt geknipt als cirkel in het event horizon
  • Accretie-schijf volledig zelfgebouwd: meerdere lagen ellipsen, dust streaks, hot spot, ambient glow en photon ring — allemaal in canvas
  • De foto van Sanne fadet in via een custom vignette zodat de randen vloeiend overgaan in de schijf

Rocket scrollbar

  • Raket-SVG vervangt de native scrollbar en rijdt langs de rechterkant van de pagina
  • Schaalflammen gloeien oranje op bij scrollen (.burning class)
  • Track toont voortgang via een CSS custom property --scroll-pct
  • Native scrollbar verborgen via scrollbar-width: none en ::-webkit-scrollbar

Typing easter egg

  • Typ STARS op het toetsenbord → meteor-shower animatie verschijnt
  • 10 shooting stars geanimeerd met CSS @keyframes
  • Sequentie-tracking via eenvoudig index-patroon (zelfde structuur als de KRAFTWERK easter eggs)

Code highlight: de black hole easter egg

Na 10 klikken op het zwarte gat wordt een canvas-gebaseerde accretie-schijf getekend. De schijf bestaat uit meerdere lagen ellipsen met licht fluctuerende opacity, dust streaks en een photon ring. Na 20 klikken fadet een foto van Sanne in het event horizon: 

// easter-egg.js — click teller
blackHole.addEventListener('click', () => {
  clicks++;
  if (clicks === 10) {
    openEgg();          // start canvas animatie
  } else if (clicks >= 20) {
    clicks = 0;
    revealSanne();     // foto in event horizon
  }
});

function drawSanne(cx, cy, R) {
  if (sanneAlpha < 1) sanneAlpha = Math.min(1, sanneAlpha + 0.012);
  eggCtx.save();
  eggCtx.globalAlpha = sanneAlpha;
  eggCtx.beginPath();
  eggCtx.arc(cx, cy, R * 0.92, 0, PI2);
  eggCtx.clip();                        // knipt cirkel uit
  eggCtx.drawImage(sanneImg, cx-r, cy-r, r*2, r*2);
  eggCtx.restore();
}

Code highlight: cursor modi + touchscreen detectie

Het cursor-systeem detecteert automatisch of de bezoeker een touchscreen heeft via matchMedia. Op touch-apparaten wordt het systeem volledig uitgeschakeld zodat de native cursor terugkomt. Op desktop wisselt de gebruiker met toetsen 1–5: 

// cursor.js — touchscreen detectie
const mq = window.matchMedia('(pointer: coarse)');

function applyPointerType(isCoarse) {
  if (isCoarse) {
    disable();      // native cursor terug
  } else {
    setMode('normal');
    enable();
  }
}

applyPointerType(mq.matches);
mq.addEventListener('change', e => applyPointerType(e.matches));

// Keybinds: 1=astronaut, 2=comet, 3=blackhole, 4=laser, 5=supernova
const KEY_MAP = {
  '1': 'astronaut', '2': 'comet', '3': 'blackhole',
  '4': 'laser',    '5': 'supernova', 'Escape': 'normal'
};

Code highlight: rocket scrollbar

De raket schrijft zijn positie via een CSS custom property naar het track-element. De ::after pseudo op de track groeit van onderaf mee, zodat je een "gelanceerde route" ziet: 

/* rocket-scrollbar.css */
#rocket-track::after {
  content: '';
  position: absolute;
  bottom: 0;
  height: calc(100% - var(--scroll-pct, 0%));
  background: linear-gradient(to top, rgba(235,93,52,0.55), rgba(110,231,255,0.3));
}

// rocket-scrollbar.js — schrijft de waarde
const pct = maxScroll > 0 ? scrollTop / maxScroll : 0;
track.style.setProperty('--scroll-pct', (pct * 100).toFixed(2) + '%');

Uitdagingen en wat ik ervan leerde

⚡ Uitdaging

Vijf canvas-systemen in één JS-file zonder dat ze elkaar beïnvloeden. Elke modus heeft zijn eigen particle array en draw-logica.

✓ Geleerd

requestAnimationFrame goed binnenshouden: één centrale loop met een rafRunning flag voorkomt meerdere loops tegelijk bij moduswissel.

⚡ Uitdaging

De accretie-schijf ziet er anders uit op elk schermformaat omdat de black hole CSS-element van grootte verandert.

✓ Geleerd

getBoundingClientRect() per frame aanroepen geeft altijd de actuele grootte en positie — ook na scroll of resize.

⚡ Uitdaging

Samenwerken in één codebase zonder merge-conflicten. Iedereen had een eigen sectie, maar gedeelde styles liepen door elkaar.

✓ Geleerd

Per feature een eigen CSS-bestand. Naamgeving afspreken voordat je begint. Bij een langere hackathon zou ik eerder een CSS custom properties file willen.

Nieuwe inzichten

01

Canvas als creatief tool

De accretie-schijf is het eerste stuk canvas-werk dat ik echt trots op ben. Canvas is veel expressiever dan ik dacht — zeker in combinatie met requestAnimationFrame en layered transparante ellipsen.

02

pointer: coarse als accessibility check

Touchscreen-gebruikers hebben geen cursor, dus een custom cursor systeem is voor hen juist verstoringsvol. matchMedia('(pointer: coarse)') laat je dit live detecteren — inclusief wanneer iemand een muis aansluit.

03

animation-timeline: view() voor parallax

CSS scroll-driven animations zijn rijp voor productiegebruik. De stagebegeleiders-afbeeldingen bewegen soepel met de pagina mee zonder één regel JavaScript — en prefers-reduced-motion schakelt ze netjes uit.

04

Samen bouwen vraagt om structuur

Eén week, vier mensen, één codebase. Zonder afspraken over naamgeving en bestandsstructuur loop je elkaar voor de voeten. Goede afspraken vooraf zijn sneller dan opruimen achteraf.

Reflectie

De hackathon was anders dan ik had verwacht — in positieve zin. Je denkt van tevoren dat je een week hebt en dat er dus tijd genoeg is. In de praktijk is een week razend snel als je ook nog content moet schrijven, een concept moet uitdenken en een team moet afstemmen.

Mijn highlight was de accretie-schijf. Het begon als een simpel "klik-10x-easter-egg" maar groeide uit tot iets waarvoor ik echt tijd heb genomen om goed te maken: de lichtfysica van een zwart gat proberen te vertalen naar canvas-ellipsen, met de juiste kleurnuances en pulse-timing. Dat Sanne's gezicht dan ook nog verschijnt na 20 klikken maakte het compleet.

Ik heb ook geleerd dat het cursor-systeem meer complexiteit had dan ik van tevoren inschat. Vijf modi, touchscreen-detectie, toast-notificaties, keybinds — het leek klein maar had veel randgevallen. Dat is misschien wel de grootste les van de hackathon: scope inschatten is een vaardigheid die je moet oefenen.
